# PRD Template

```markdown
# [Feature] PRD

| [Feature] PRD | | [Summary] |
|:---|:---|:---|
| **Author**: [Name] **Audience**: Engineering | **Status**: Draft **Created**: [Date] | **Visibility**: Internal |

## Completion Checklist
- [ ] Problem Statement
- [ ] Objective & Scope
- [ ] Functional Requirements
- [ ] Interface Contracts
- [ ] Test Expectations

## Problem
**Current Process**: | **Users**: | **Pain Points**: | **Importance**:

## Objective & Scope
**Objective**: [Single measurable goal]
**Done looks like**: [Concrete outcome]

### In-scope
- [What to build]

### Not-in-scope
- [What NOT to build]

## Requirements
| Priority | Requirement | Verification |
|:---------|:------------|:-------------|
| P0       | [Must have] | [command/test] |
| P1       | [Should have] | [command/test] |

## Interface Contracts
| Boundary | Input | Output | Error |
|:---------|:------|:-------|:------|

## Test Expectations
| Requirement | Test File | Description | Assertion |
|:------------|:----------|:------------|:----------|

## Assumptions
## Risks & Mitigations
## Context
- Key files: [paths]
- Patterns to follow: [examples]
```

Mark checklist items as sections are drafted. Every requirement needs a Verification column — machine-checkable command, test, or assertion.
